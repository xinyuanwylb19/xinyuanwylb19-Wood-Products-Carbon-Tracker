📖 User Guide

How to prepare data, run the tracker, and interpret results

Based on: Wei et al. (2023) · Carbon Balance and Management 18:1

⬇ Download Full Manual (.docx)

⚡ Quick Start — End-use Products Tracker

  1. 1

    Select a Data Source

    Choose the built-in Maine or US example, or upload your own production CSV (see format below).

  2. 2

    Choose Model Parameters

    Use the default parameters (Wei et al. 2023) or upload a custom parameters CSV for your region.

  3. 3

    ⚙️ Initialize Python

    Click "Initialize Python". The first load downloads the Python runtime (~50 MB, 30–90 sec). Subsequent loads within the same session are instant.

  4. 4

    ▶ Run Carbon Tracker

    Runs the full life-cycle model. Results appear in the table; the Download button activates.

  5. 5

    📊 Plot & Explore

    Use "Plot Input" and "Plot Carbon Stock" to visualise production trends and accumulated carbon pools. Hover charts for exact values.

📂 Input Data CSV Format

The production data file must be a comma-separated (.csv) file with the following columns. All numeric values must be in kg C per year.

Column Name Pool Description
Year—Calendar year (integer). Required. No gaps.
BiofuelBioenergyAnnual biofuel combustion (kg C/yr) — industrial + residential fuel wood.
BiocharCharcoalNon-energy use biochar production (kg C/yr).
ConstructionEnd-useWood in building construction — sawnwood, structural panels (kg C/yr).
ExteriorEnd-useExterior products — railway ties, wood decking, dock materials (kg C/yr). No recycling.
HouseholdEnd-useHome application — furniture, flooring, interior millwork (kg C/yr).
Graphic PaperEnd-usePrinting & writing paper, newsprint (kg C/yr). Service half-life γ = 6 yr.
Other PaperEnd-usePackaging, cardboard, industrial paperboard (kg C/yr). Service half-life γ = 1 yr.
Household PaperEnd-useTissue, toweling, sanitary paper (kg C/yr). Service half-life γ = 0.5 yr. No recycling.

🌲 Carbon Pools & Flux Equations

The tracker maintains three main carbon sub-pools, governed by the equations from Wei et al. (2023):

① End-use Products

Carbon remaining in in-use wood products (construction, furniture, paper).

Eq. 2–3: Gaussian disposal
ρwp = α/e√(2π) · exp(−β(tw−γ)²/γ)

② Landfill Pool

Disposed wood undergoing slow decomposition in landfills.

Eq. 5: Log-normal decay
ρlf = ξ · ln(tl) / (ω·√(2π))

③ Biochar / Charcoal

Recalcitrant carbon from biofuel combustion and biochar production.

Eq. 1: Pool-size decay
ρcha = τ + σ · ln(Ccha)

Recycling pathway (Eq. 4): r = λ + μ · ln(k), where k = years since initial year. Recycling diverts disposed wood from landfill, increasing effective carbon retention.

🐙 Hosting Files on GitHub

All supplementary files (this manual, example CSVs, images) should live in your repository alongside index.html. GitHub Pages will serve them automatically.

Recommended folder structure

your-repo/
├── index.html
├── assets/
│   ├── img/
│   │   ├── Logo_USDA.png
│   │   ├── Logo_NSRC.png
│   │   ├── Logo_WGL.png
│   │   └── WPs_Carbon Flow.png
│   └── docs/
│       └── WPsCT_UserManual.docx   ← place this file here
└── data/
    ├── Maine_example.csv
    └── US_example.csv

Uploading via GitHub browser interface

  1. 1.Open your repository on GitHub.com → click Add file → Upload files.
  2. 2.Navigate to assets/docs/ (create the folder by typing it in the path bar).
  3. 3.Drag and drop WPsCT_UserManual.docx → click Commit changes.
  4. 4.The file is now available at:
    https://<username>.github.io/<repo>/assets/docs/WPsCT_UserManual.docx
File size limits: GitHub browser upload limit is 25 MB. This manual is ~25 KB — well within limits. For large datasets (>25 MB), use Zenodo or OSF and link to the DOI.
